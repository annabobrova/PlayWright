 CONCEPTS TO STUDY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[ ] pytest.mark.parametrize
      - Run one test function with multiple sets of inputs

[ ] Fixtures with yield vs try/finally
      - Two ways to write teardown — know both and when to use each

[ ] page.wait_for_selector vs expect()
      - expect() auto-waits and retries; wait_for_selector waits for DOM presence
      - Prefer expect() for assertions in Playwright

[ ] page.request vs the requests library
      - page.request shares the Playwright browser context (cookies, auth)
      - requests library is independent — know why this project uses page.request

[ ] Docker — run the tests in a container
      - Command: docker-compose up
      - Tests should pass in CI, not just on your machine

[ ] allure-pytest report
      - More powerful report format used in many companies
      - Install: pip install allure-pytest
      - Run: python3 -m pytest --alluredir=allure-results
